# 🚀 快速开始指南

## 欢迎使用"南粤风炊火"野炊教学应用！

本指南将帮助您在5分钟内运行这个Android应用。

---

## 📋 准备工作清单

### ✅ 必需软件
- [ ] Android Studio (最新版)
- [ ] JDK 8 或更高版本
- [ ] 10-11寸Android平板（或模拟器）

### ✅ 确认事项
- [ ] 平板Android版本 ≥ 7.0
- [ ] USB调试已开启
- [ ] 有足够的存储空间（约500MB）

---

## 🔧 第一步：配置Android Studio

### 1. 打开项目
```
1. 启动 Android Studio
2. 选择 "Open an Existing Project"
3. 浏览到项目文件夹 "CampcookingAR"
4. 点击 "OK"
```

### 2. 等待同步完成
- Gradle会自动下载依赖
- 首次同步可能需要5-10分钟
- 请保持网络连接畅通

### 3. 配置SDK路径（如需要）
如果提示找不到SDK，创建 `local.properties` 文件：
```properties
sdk.dir=C\:\\Users\\你的用户名\\AppData\\Local\\Android\\Sdk
```

---

## 📱 第二步：准备设备

### 方式A：使用真实平板（推荐）

1. **开启开发者选项**
   ```
   设置 → 关于平板 → 连续点击"版本号"7次
   ```

2. **开启USB调试**
   ```
   设置 → 开发者选项 → USB调试（打开）
   ```

3. **连接平板**
   ```
   使用USB线连接平板到电脑
   平板上点击"允许USB调试"
   ```

### 方式B：使用模拟器

1. 在Android Studio中点击 "AVD Manager"
2. 点击 "Create Virtual Device"
3. 选择平板设备（如 Pixel C）
4. 选择系统镜像（推荐API 34）
5. 点击 "Finish" 创建

---

## ▶️ 第三步：运行应用

### 方法1：点击运行按钮
```
1. 在Android Studio顶部找到绿色三角形按钮 ▶️
2. 确认选择了正确的设备
3. 点击运行按钮
4. 等待编译和安装（首次约2-3分钟）
```

### 方法2：使用命令行
```bash
# Windows
cd CampcookingAR
gradlew.bat installDebug

# Mac/Linux
cd CampcookingAR
./gradlew installDebug
```

## 🐛 常见问题解决

### ❌ 问题1：Gradle同步失败
**症状：** "Gradle sync failed"

**解决方案：**
```
1. 检查网络连接
2. File → Invalidate Caches → Invalidate and Restart
3. 或手动下载Gradle 8.2
```

### ❌ 问题2：找不到设备
**症状：** "No devices found"

**解决方案：**
```
1. 重新插拔USB线
2. 在平板上重新授权USB调试
3. 运行命令：adb devices 检查连接
```

### ❌ 问题3：编译错误
**症状：** "Build failed"

**解决方案：**
```
1. Build → Clean Project
2. Build → Rebuild Project
3. 确认SDK API 34已安装
```

### ❌ 问题4：应用显示不正常
**症状：** 布局错乱或文字过小

**解决方案：**
```
1. 确保设备处于横屏模式
2. 检查设备屏幕尺寸（建议10-11寸）
3. 重启应用
```
